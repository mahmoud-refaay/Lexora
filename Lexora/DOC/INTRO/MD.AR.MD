# Lexora (ميزان)

## نظام إدارة مكتب محاماة

الإصدار: 1.0 مسودة

حالة المشروع: مرحلة التخطيط

التحقق من متطلبات العمل: في انتظار مراجعة من محامين وموظفين داخل مكاتب محاماة حقيقية.

---

## 1. رؤية المشروع

Lexora هو نظام احترافي لإدارة مكاتب المحاماة، مصمم لمساعدة مكاتب المحاماة والمحامين المستقلين على إدارة العمليات القانونية اليومية من منصة مركزية واحدة.

الاسم العربي: ميزان

الاسم الإنجليزي: Lexora

نوع المشروع: ASP.NET Core Web API

المعمارية: Clean Architecture

المستخدمون الأساسيون: مدير النظام، المحامون، السكرتارية، المحاسبون.

هدف النظام هو تقليل الاعتماد على الورق، تحسين متابعة القضايا، تنظيم المستندات القانونية، إدارة الجلسات والمواعيد، وتوفير رؤية أوضح للمدفوعات وأداء المكتب.

---

## 2. نطاق العمل التجاري

نطاق العمل الأساسي هو إدارة الممارسة القانونية داخل مكتب محاماة.

يجب أن يعكس النظام طريقة العمل الواقعية داخل المكتب:

1. يتواصل العميل مع المكتب.
2. يتم إنشاء ملف للعميل.
3. يتم فتح قضية أو ملف قانوني للعميل.
4. يتم ربط المستندات والملاحظات والمهام والمدفوعات والجلسات بالقضية.
5. يتابع المكتب مواعيد الجلسات، بيانات المحكمة، تحديثات القضية، والإجراءات المطلوبة.
6. يتم تتبع المدفوعات والأرصدة على مستوى العميل والقضية.
7. تساعد التقارير المكتب على فهم ضغط العمل، الإيرادات، حالة القضايا، وأداء الموظفين.

يجب تصميم النظام حول مفاهيم مكتب المحاماة الثابتة، وليس حول الشاشات فقط. هذا يجعل المشروع قابلا للتوسع لاحقا لإضافة الإشعارات، مزامنة التقويم، OCR، مساعد ذكي، تطبيق موبايل، أو نسخة SaaS متعددة المكاتب.

## 3. الأهداف الرئيسية

- تنظيم عمليات مكتب المحاماة.
- تقليل الورق والمتابعة اليدوية.
- متابعة العملاء، القضايا، الجلسات، المهام، المستندات، والمدفوعات.
- تخزين المستندات القانونية بشكل آمن مع بيانات وصفية قابلة للبحث.
- تحسين الإنتاجية من خلال التذكيرات والإشعارات.
- توفير تقارير واضحة للقضايا، الجلسات، العملاء، المدفوعات، وأداء الموظفين.
- تتبع العمليات الحساسة من خلال Audit Logs.
- بناء Backend معياري قابل للتوسع مستقبلا.

## 4. المفاهيم الأساسية

### العميل

شخص أو شركة أو جهة تحصل على خدمة قانونية من المكتب.

بيانات العميل قد تشمل الاسم الكامل، الرقم القومي، رقم الهاتف، البريد الإلكتروني، العنوان، الملاحظات، والقضايا المرتبطة.

### القضية

ملف قانوني يديره المكتب لصالح عميل. يمكن أن تحتوي القضية على جلسات، مستندات، ملاحظات، مهام، مدفوعات، وتغييرات حالة.

### الجلسة

موعد محكمة أو إجراء قانوني مرتبط بقضية. الجلسات حساسة للوقت ويجب أن تدعم التذكيرات.

### المستند

ملف قانوني مرتبط بعميل أو قضية، مثل العقود، التوكيلات، المذكرات القانونية، مستندات المحكمة، الأحكام، والمرفقات.

### المهمة

عنصر عمل داخلي يتم تعيينه لمستخدم، وغالبا يكون مرتبطا بقضية أو جلسة أو عميل أو مستند.

### الدفعة

عملية مالية مرتبطة بعميل أو قضية. تساعد المدفوعات على تتبع الإيرادات، الرصيد المتبقي، والتاريخ المالي.

### الإشعار

تنبيه أو تذكير خاص بالجلسات، المهام، القضايا الجديدة، متابعة المدفوعات، أو أي إجراء مهم داخل المكتب.

### سجل التدقيق

تسجيل للعمليات المهمة داخل النظام مثل تسجيل الدخول، إنشاء قضية، تعديل دفعة، حذف مستند، أو تغيير صلاحيات مستخدم.

## 5. أدوار المستخدمين

### Admin

مدير النظام وله صلاحيات كاملة.

الصلاحيات:

- وصول كامل للنظام.
- إدارة المستخدمين.
- إدارة الأدوار.
- إدارة الصلاحيات.
- إعدادات النظام.
- التقارير.
- سجلات التدقيق.

### Lawyer

مستخدم قانوني مسؤول عن العمل على القضايا.

الصلاحيات:

- إدارة العملاء المعينين له.
- إدارة القضايا المعينة له.
- إدارة الجلسات.
- إدارة المستندات القانونية.
- إدارة ملاحظات القضايا.
- إدارة المهام.
- عرض التقارير المرتبطة بعمله.

### Secretary

مستخدم تشغيلي مسؤول عن التنسيق والمواعيد.

الصلاحيات:

- إضافة وتحديث العملاء.
- جدولة الجلسات.
- إدارة المواعيد.
- رفع المستندات.
- إنشاء المهام.
- متابعة الجلسات القادمة والتذكيرات.

### Accountant

مستخدم مالي مسؤول عن المدفوعات والتقارير المالية.

الصلاحيات:

- إضافة المدفوعات.
- تعديل ملاحظات الدفع.
- عرض سجل المدفوعات.
- متابعة الأرصدة المستحقة.
- عرض التقارير المالية.
- متابعة الإيراد الشهري.

## 6. الوحدات الأساسية

الخصائص:

- تسجيل الدخول.
- تسجيل الخروج.
- تغيير كلمة المرور.

- Refresh Tokens.

## 7. مسارات العمل العامة

### مسار عميل وقضية جديدة

1. يقوم السكرتير أو المحامي بإنشاء ملف عميل.
2. يقوم المحامي بإنشاء قضية للعميل.
3. يتم إدخال بيانات القضية، المحكمة، الخصم، والمحامي المسؤول.
4. يتم رفع المستندات وربطها بالقضية.
5. يتم إنشاء الجلسات والمهام.
6. يتم تسجيل المدفوعات عند استلامها.
7. يتم تتبع تقدم القضية من خلال Timeline، الملاحظات، الجلسات، والمهام.
8. يتم إغلاق القضية عند الانتهاء ثم أرشفتها عند الحاجة.

### مسار الجلسة

1. يقوم المستخدم بجدولة جلسة لقضية نشطة.
2. يحفظ النظام المحكمة، التاريخ، الوقت، القاضي، والملاحظات.
3. ينشئ النظام تذكيرا للجلسة القادمة.
4. بعد الجلسة يضيف المحامي النتيجة والإجراء التالي.
5. إذا تم التأجيل يمكن إنشاء موعد جلسة جديد.

### مسار الدفع

1. يختار المحاسب العميل ويمكنه اختيار القضية.
2. يسجل المحاسب المبلغ، التاريخ، طريقة الدفع، والملاحظات.
3. يحدث النظام سجل المدفوعات وإحصائيات الإيراد.
4. تظهر العملية في التقارير المالية.
5. أي تعديل أو إلغاء يتم تسجيله في Audit Logs.

## 8. المعمارية التقنية

### Backend

- ASP.NET Core 8 Web API.
- C#.
- SQL Server.
- Dapper ORM.
- Stored Procedures.
- Swagger OpenAPI.
- FluentValidation.
- Serilog.

### طبقات Clean Architecture

- Presentation Layer: Controllers، Endpoints، Request/Response Models، وAuthentication Middleware.
- Application Layer: Use Cases، Services، DTOs، Validation، Authorization Policies، وInterfaces.
- Domain Layer: Entities، Enums، Domain Rules، وDomain Errors.
- Infrastructure Layer: Database Access، Dapper Repositories، Stored Procedures، File Storage، Email/SMS Providers، وLogging Integrations.

### Design Patterns

- Repository Pattern.
- Unit of Work.
- Dependency Injection.
- CQRS اختياري عند الحاجة لفصل القراءة عن الكتابة.
- Options Pattern للإعدادات.

- المحاسب يصل للوحدات المالية، وليس للملاحظات القانونية الحساسة إلا عند السماح.
- السكرتارية تدير المواعيد والمستندات ولا تدير الأدوار أو إعدادات النظام.
- كل عملية حساسة يجب تسجيلها مع User ID، Action، Entity، Entity ID، Date، وIP Address إن توفر.
